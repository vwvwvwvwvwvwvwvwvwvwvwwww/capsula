

```bash
npm install
cp .env.example .env
# отредактируйте .env при необходимости (пути к БД, каталогу и SMTP)
npm start
```

Сервер слушает порт из переменной **`PORT`** (по умолчанию **3333**). Откройте в браузере `http://localhost:3333`.

## Полезные команды

| Команда | Назначение |
|--------|------------|
| `npm start` | Запуск сервера |
| `npm run create-admin` | Создать учётную запись администратора |
| `npm run test-mail` | Проверить SMTP и отправить тестовое письмо |
| `npm run import-lime` | Импорт каталога (скрипт в `scripts/`) |

Подробности деплоя на **Railway** — в файле [`RAILWAY_DEPLOY.md`](RAILWAY_DEPLOY.md).

## Почтовые уведомления

Письма отправляются через любой SMTP-провайдер. Проверка: `npm run test-mail`. Для Яндекса нужен **пароль приложения** (не обычный пароль) и включённый доступ «Почтовые программы» в настройках почты. Если SMTP недоступен, письма сохраняются в `data/mail-outbox/`, а предзаказ всё равно попадает в базу.

## Что не коммитить

- **`.env`** — секреты и локальные настройки (в репозитории есть только **`.env.example`**).
- **`data/*.db`**, **`data/products.json`** — локальные данные и копия каталога (при первом запуске `products.json` создаётся из `lib/catalog-default.json`).

## Структура (кратко)

- `server.mjs` — HTTP API и раздача статики
- `lib/` — хранилище SQLite, каталог, почта, оплата
- `data/` — локальная БД и данные (часть путей в `.gitignore`)
- `css/`, `js/`, `*.html` — страницы витрины и админки
